Security & data handling

SafePrompt is built for one outcome: protect yourself and your customers when using LLMs. Our security model is intentionally simple:

  • Local-only processing. Detection and tokenization run in your browser.
  • No data collection. We don’t run accounts or receive your prompt/response content.
  • Transparent permissions. The extension only operates on supported LLM sites.

Data flow (high level)

  • You write a prompt.
  • SafePrompt can warn you if it detects likely PII (warn is free).
  • When you click Secure Send, SafePrompt replaces matched values with tokens (e.g. [[SP_EMAIL_1]]).
  • Only the sanitized prompt is sent to the LLM website.
  • You can restore locally when you choose.

Local storage

To support restore, SafePrompt stores token mappings locally in a per‑tab “vault” using Chrome session storage. The vault expires automatically (about 30 minutes), and you can clear it at any time.

Extension permissions (why they exist)

  • storage: stores settings and local token mapping.
  • activeTab / tabs: enables per‑tab vault behavior.
  • Host permissions: limited to supported sites (ChatGPT/Claude/Gemini/Grok/xAI).

Hard boundaries

  • No accounts / login.
  • No servers required for the extension to function.
  • No prompt text or response text is sent to SafePrompt servers.
