Privacy Policy

Last updated: 2026-01-23

SafePrompt is built to help you protect yourself and your customers when working with LLMs. Our privacy approach is simple: we do not collect your prompts or personal data.

Summary

  • No data collection: SafePrompt does not run accounts, does not send your prompt/response text to SafePrompt servers, and does not sell data.
  • Local processing: detection and tokenization run in your browser.
  • Local storage only: token mappings are stored per-tab in Chrome session storage and expire automatically.

What SafePrompt processes

To sanitize a prompt, the extension reads the text you are about to send and detects patterns that may represent PII or secrets (for example: emails, phone numbers, IDs, API keys). It then replaces matched values with SafePrompt tokens (for example: [[SP_EMAIL_1]]) before the prompt is sent to the LLM website.

What SafePrompt stores

SafePrompt stores a token mapping locally so it can restore original values when you choose Reveal/Restore. This mapping is stored in a per-tab “vault” using Chrome’s session storage and expires automatically (about 30 minutes). You can clear the vault at any time.

What SafePrompt does not collect

  • Prompt text or response text sent to SafePrompt servers
  • Account information (there are no accounts)
  • Browsing history (beyond the extension operating on supported LLM sites)

Extension permissions

SafePrompt requests the minimum permissions needed to work:

  • storage: to store settings and the local per-tab vault.
  • activeTab / tabs: to operate on the current tab and support per-tab storage.
  • Host permissions: limited to supported LLM sites (e.g. chatgpt.com, claude.ai, gemini.google.com, grok.com, x.ai).

This website

This landing page is a static site. We do not include analytics scripts or tracking pixels here. GitHub Pages may log standard access information (for example, IP address and user agent) as part of hosting the website.
