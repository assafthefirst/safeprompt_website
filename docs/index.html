How it works

You keep working as usual. SafePrompt adds a lightweight “secure send” step so sensitive information doesn’t leave your browser.

1) Detect & replace

Finds common PII and secrets (emails, phone numbers, IDs, API keys, tokens) and replaces them with SafePrompt tokens like [[SP_EMAIL_1]].

2) Send sanitized text

Only the sanitized prompt is sent to the LLM. The original values stay on your device.

3) Restore locally

When you click Reveal/Restore, SafePrompt swaps tokens back to originals locally — using a per‑tab session vault that expires automatically.

Coverage

Designed for client-facing SMB teams. The goal is simple: catch the sensitive data you didn’t mean to share.

PII

Emails, phone numbers (US/IL), IDs (incl. multiple regions), dates of birth (context-aware warnings), and select address patterns.

Secrets

Common credentials like JWTs, API keys, GitHub/Slack tokens, Stripe secrets, and private key blocks.

High-risk identifiers

Credit cards (validated), IBAN (checksum), SWIFT/BIC (keyword), IP/MAC, GPS coordinates, and wallet-like addresses.

FAQ

Short answers to the most common questions from teams.

Do you collect my prompts or responses?

No. SafePrompt performs detection and tokenization locally in your browser. We don’t run accounts, backends, or analytics telemetry for the extension.

Where is the mapping stored?

The token mapping is stored locally in a per‑tab session vault (Chrome session storage), and expires automatically (about 30 minutes). You can also clear it anytime.

Which sites are supported?

Currently: ChatGPT (chatgpt.com), Claude (claude.ai), Gemini (gemini.google.com), Grok (grok.com), and xAI (x.ai).

What if you warn by mistake (false positives)?

Warnings are conservative and context-aware, and Secure Send is designed to be reliable. You can always continue unprotected once if needed.

Is this affiliated with OpenAI / Anthropic / Google / xAI?

No. SafePrompt is an independent product and is not affiliated with those companies.
